# Tienda de Comercio Electrónico - Programación Web II (Semana 8)

Este proyecto representa una aplicación web Fullstack dinámica que evoluciona hacia un entorno de desarrollo colaborativo y profesional utilizando **Git y GitHub**. El sistema integra la gestión avanzada de ramas, el control de versiones, la revisión de código entre pares y el seguimiento de incidencias, garantizando la calidad, seguridad y estabilidad del código base del e-commerce.


## Características Implementadas

1. **Estrategia de Ramificación (Feature Branch Workflow):** Aislamiento absoluto de nuevas funcionalidades en ramas independientes (como `feature/gestion-pedidos`), impidiendo modificaciones directas sobre la rama principal (`main`) y protegiendo el entorno estable del sistema.
2. **Solicitudes de Extracción (Pull Requests) y Revisión de Código:** Implementación de Pull Requests (PRs) como mecanismo formal para proponer e integrar cambios al proyecto. Incluye un proceso de revisión técnica (*Code Review*) entre colaboradores con comentarios en línea y sugerencias de optimización antes de fusionar (*merge*) el código.
3. **Gestión e Historial de Incidencias (GitHub Issues):** Documentación y seguimiento estructurado de tareas, bugs y nuevas características (como la integración de la pasarela de pago segura y sincronización de stock en MySQL), facilitando el debate técnico, asignación de responsabilidades y categorización mediante etiquetas (*labels*).
4. **Buenas Prácticas de Control de Versiones:** Aplicación de mensajes de commit semánticos y descriptivos (`feat:`, `fix:`, `docs:`), verificación mediante pruebas locales previas a cada confirmación y resguardo de datos sensibles o credenciales de base de datos (`conexion.php`) en el entorno remoto.
